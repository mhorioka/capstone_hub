# Feature Spec: プロジェクト管理

## Overview
ユーザーが食品メーカー市場参入プロジェクトを作成・閲覧・削除する機能。

**Route**: `/`（一覧） / `/projects/[id]`（ダッシュボード）

---

## 1. プロジェクト一覧ページ (`/`)

### 表示要件
- [ ] 既存プロジェクトをカード形式で一覧表示する
- [ ] 各カードに表示する情報:
  - プロジェクト名
  - 対象市場
  - 食品カテゴリ
  - 目標期日
  - 最終更新日時
  - 3セクション（市場調査/競合分析/GTM）それぞれの完成度ステータスバッジ
- [ ] プロジェクトが0件の場合は空状態メッセージを表示する
- [ ] 「新規プロジェクト作成」ボタンを表示する

### インタラクション
- [ ] カードをクリックすると `/projects/[id]` に遷移する
- [ ] 「新規プロジェクト作成」ボタンクリックで作成フォームを表示する（モーダルまたは展開エリア）

---

## 2. プロジェクト作成

### 入力フォーム
| フィールド | 型 | 必須 |
|---|---|---|
| プロジェクト名 | text | ✓ |
| 対象市場 | text | ✓ |
| 食品カテゴリ | text | ✓ |
| 目標期日 | date | ✓ |

### バリデーション
- [ ] **いずれかの**必須フィールドが空の場合、送信ボタンを無効化する（一部入力済みでも全必須フィールドが埋まるまで送信不可）
- [ ] 送信後フォームをリセットする

### 保存動作
- [ ] ID: `uuid()` で生成
- [ ] `createdAt` / `updatedAt`: 作成時刻
- [ ] 各セクション（marketResearch / competitiveAnalysis / gtmPlan）は空の初期値で作成
- [ ] localStorage に保存後、作成したプロジェクトの `/projects/[id]` へ遷移する

---

## 3. プロジェクトダッシュボード (`/projects/[id]`)

### 表示要件
- [ ] プロジェクト名・対象市場・食品カテゴリ・目標期日を表示する
- [ ] 3セクションのカード（タイトル・ステータス・「開く」ボタン）を表示する
- [ ] 全体完成度の視覚的表現（プログレスバーなど）を表示する

### インタラクション
- [ ] 各セクションカードの「開く」ボタンで対応ページへ遷移する
- [ ] プロジェクト削除ボタンを表示し、確認ダイアログの後に削除・一覧へリダイレクト

---

## 4. データ永続化

- [ ] プロジェクト一覧インデックスは `capstone_v1_projects_index` に保存
- [ ] 各プロジェクトは `capstone_v1_project_[id]` に保存
- [ ] ページリロード後も一覧・詳細が復元される

---

## 5. エラーケース

| ケース | 期待動作 |
|---|---|
| 存在しない `/projects/[id]` にアクセス | 「プロジェクトが見つかりません」メッセージを表示し、一覧へ誘導 |
| localStorage が満杯 | エラーメッセージをトースト等で通知 |
