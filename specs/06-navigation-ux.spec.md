# Feature Spec: ナビゲーション & UX

## Overview
サイドバーナビゲーション、完成度インジケーター、自動保存、言語切り替えの横断的 UX 仕様。

---

## 1. レイアウト構造

```
┌─────────────────────────────────────────┐
│  Header (固定)                           │
│  [アプリ名]              [JA / EN toggle] │
├────────────┬────────────────────────────┤
│  Sidebar   │  Main Content Area          │
│  (240px)   │  (可変幅、最大 1280px)        │
│            │                            │
│  ナビリンク │  ページコンテンツ             │
│  + ステータス│                            │
└────────────┴────────────────────────────┘
```

### 表示要件
- [ ] Header は全ページで固定表示する
- [ ] Sidebar はプロジェクト詳細ページ（`/projects/[id]/*`）でのみ表示する
- [ ] メインコンテンツエリアは Sidebar の幅に応じてパディングを調整する

---

## 2. サイドバーナビゲーション

### 表示項目
- [ ] ダッシュボード（`/projects/[id]`）
- [ ] 市場調査（`/projects/[id]/market`）
- [ ] 競合分析（`/projects/[id]/competitive`）
- [ ] GTMプラン（`/projects/[id]/gtm`）
- [ ] レポート（`/projects/[id]/report`）

### セクション完成度インジケーター
各ナビリンクに StatusBadge を表示:

| Status | 表示 | 色 |
|---|---|---|
| `empty` | 未入力 | グレー |
| `in_progress` | 入力中 | イエロー/アンバー |
| `done` | 完了 | グリーン |

### インタラクション
- [ ] 現在のページリンクをアクティブスタイルで強調表示する
- [ ] リンクはクリックで対応ページへ遷移する

---

## 3. 自動保存

### 動作仕様
- [ ] テキスト入力・選択・リスト変更のたびに 300ms の debounce タイマーをリセットする
- [ ] debounce 完了後、現在のプロジェクト状態全体を localStorage に書き込む
- [ ] 保存ステータスをヘッダーまたはサイドバー付近に表示する:
  - 入力中: 表示なし（または点滅インジケーター）
  - 保存中: 「保存中...」
  - 保存完了: 「保存済み」（2秒後に非表示）
  - 保存失敗: 「保存に失敗しました」（エラー色で常時表示）

### 実装メモ
- debounce は各入力フィールドではなく、ProjectContext のディスパッチ後にかける
- 保存状態は `isDirty` フラグと組み合わせて管理する

---

## 4. 言語切り替え

### 表示要件
- [ ] Header 右端に「JA / EN」トグルを常時表示する
- [ ] 現在の言語を視覚的に強調表示する（太字、背景色等）

### 動作仕様
- [ ] トグルクリックで即時に全 UI 文字列を切り替える（ページリロードなし）
- [ ] 選択言語を `capstone_v1_language` に保存する
- [ ] ページリロード後も選択した言語を維持する

### 対応言語
| コード | 言語名 |
|---|---|
| `ja` | 日本語（デフォルト） |
| `en` | English |

---

## 5. アクセシビリティ

- [ ] フォームの各入力フィールドに `<label>` を関連付ける
- [ ] ボタンに `aria-label` または適切なテキストを設定する
- [ ] 色のみに依存しないステータス表現（テキストラベルを併用）
- [ ] キーボードフォーカスが視覚的に確認できる（focus-visible スタイル）
- [ ] コントラスト比: テキストと背景で 4.5:1 以上

---

## 6. レスポンシブ

- [ ] デスクトップ優先（1280px 以上）を基本とする
- [ ] 1280px 未満ではサイドバーを折りたたみ可能にする（Phase 2 対応で可）
- [ ] MVP では横スクロールが出ても許容する
