# Feature Spec: レポートビュー

## Overview
入力された全セクションのデータを1ページで閲覧し、ブラウザ印刷 / PDF 出力できるビュー。

**Route**: `/projects/[id]/report`

---

## 1. ページ構成

レポートは以下のセクションを縦に順番で表示する:

| 順序 | セクション |
|---|---|
| 1 | プロジェクト情報（名称・対象市場・食品カテゴリ・目標期日） |
| 2 | 市場調査 |
| 3 | 競合分析 |
| 4 | GTMプラン |

---

## 2. 各セクションの表示内容

### 2-1. プロジェクト情報
- [ ] プロジェクト名（見出し）
- [ ] 対象市場・食品カテゴリ・目標期日・レポート生成日時

### 2-2. 市場調査
- [ ] **市場概要**: 市場規模・CAGR・市場定義を表形式で表示
- [ ] **ターゲットセグメント**: セグメント一覧（名称・規模・特徴）
- [ ] **消費者インサイト**: インサイト一覧（情報源・メモ・日付）
- [ ] **規制・参入障壁**: 障壁一覧（種別・説明）
- [ ] **市場トレンド**: トレンド一覧（タイトル・説明）

### 2-3. 競合分析
- [ ] **競合プロファイル**: 競合テーブル（企業名・市場シェア・強み/弱み・製品・価格帯）
- [ ] **ポジショニングマップ**: Recharts グラフを表示（印刷時も可視）
- [ ] **SWOT分析**: 4象限グリッド表示
- [ ] **差別化ポイント**: テキスト表示

### 2-4. GTMプラン
- [ ] **ペルソナ**: ペルソナカード一覧（名称・年齢帯・職業・ペインポイント・ゴール）
- [ ] **参入戦略**: 選択チャネル一覧・価格戦略テキスト
- [ ] **施策一覧**: 施策テーブル（タイトル・説明・担当・期日）
- [ ] **フェーズ計画**: Phase 1/2/3 のマイルストーン・期間・目標
- [ ] **KPI**: KPI テーブル（指標名・目標値・測定方法）

---

## 3. 印刷 / PDF出力

### 要件
- [ ] 「印刷 / PDF出力」ボタンをページ上部に表示する
- [ ] ボタン押下で `window.print()` を呼び出す
- [ ] `@media print` スタイルで以下を制御する:
  - サイドバー・ヘッダーのナビゲーションを非表示
  - 「印刷 / PDF出力」ボタンを非表示
  - 各セクション見出し前で改ページ（`page-break-before: always`）
  - 白背景・黒テキストで印刷最適化

### 印刷レイアウト
```css
@media print {
  /* ナビゲーション非表示 */
  nav, aside, .no-print { display: none; }
  /* セクション改ページ */
  .print-section { page-break-before: always; }
  /* 最初のセクションは改ページなし */
  .print-section:first-child { page-break-before: auto; }
}
```

---

## 4. 空データの扱い

- [ ] データが未入力のサブセクションは「（未入力）」またはプレースホルダーテキストを表示する
- [ ] セクションが `empty` の場合でもセクション見出しは表示する（レポート構造の一貫性のため）

---

## 5. エラーケース

| ケース | 期待動作 |
|---|---|
| プロジェクトが存在しない | 「プロジェクトが見つかりません」を表示し一覧へ誘導 |
| 競合が0件でポジショニングマップを表示 | 「競合データがありません」メッセージを表示 |
