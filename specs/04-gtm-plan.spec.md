# Feature Spec: GTMプラン

## Overview
Go-to-Market 計画を構造化して入力・管理する。ターゲット顧客ペルソナ、参入戦略、施策、フェーズ計画、KPI を含む。

**Route**: `/projects/[id]/gtm`

---

## サブセクション一覧

| # | サブセクション | 必須フィールド（`in_progress` 判定用） |
|---|---|---|
| 1 | ターゲット顧客ペルソナ | 1件以上の `name` |
| 2 | 参入戦略 | `channels` 1件以上 OR `pricingStrategy` |
| 3 | マーケティング施策 | 1件以上の `title` |
| 4 | フェーズ計画 | 1件以上の `milestone` |
| 5 | KPI | 1件以上の `metric` |

## 2. 参入戦略

### 表示要件
- [ ] 参入チャネルをチェックボックス群で表示する（小売 / EC / B2B / 卸売 / その他）
- [ ] 価格戦略のテキストエリアを表示する

### インタラクション
- [ ] チェックボックス変更後 300ms debounce で自動保存
- [ ] テキストエリア変更後 300ms debounce で自動保存
- [ ] 複数チャネル選択可能

## セクション完成度ロジック

```
status = 'empty'
  → 全サブセクションに入力なし

status = 'in_progress'
  → personas.length > 0
    OR entryStrategy.channels.length > 0
    OR entryStrategy.pricingStrategy !== ''
    OR initiatives.length > 0
    OR phases.length > 0
    OR kpis.length > 0

status = 'done'
  → personas.length >= 1
    AND entryStrategy.channels.length >= 1
    AND entryStrategy.pricingStrategy !== ''
    AND initiatives.length >= 1
    AND phases.length >= 1
    AND kpis.length >= 1
```

## エラーケース

| ケース | 期待動作 |
|---|---|
| 必須フィールドが空で追加押下 | フィールドにエラー表示、保存しない |
| フェーズ番号が重複 | 確認ダイアログを表示、OK で上書き・キャンセルで中断 |
| 終了日 < 開始日 | フィールドにエラー表示 |
