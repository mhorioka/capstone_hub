# Feature Spec: 市場調査

## Overview
食品メーカーの市場参入計画に必要な市場調査データを構造化して入力・管理する。

**Route**: `/projects/[id]/market`

---

## サブセクション一覧

| # | サブセクション | 必須フィールド（`in_progress` 判定用） |
|---|---|---|
| 1 | 市場概要 | `definition` |
| 2 | ターゲットセグメント | 1件以上の `name` |
| 3 | 消費者インサイト | 1件以上の `notes` |
| 4 | 規制・参入障壁 | 1件以上の `description` |
| 5 | 市場トレンド | 1件以上の `title` |

---

## 1. 市場概要

### 表示要件
- [ ] 市場規模・CAGR・市場定義の入力フィールドを表示する

### インタラクション
- [ ] 各フィールドへの入力後 300ms の debounce で自動保存する
- [ ] `definition` が入力されると section status が `in_progress` になる
- [ ] 市場規模・CAGR・市場定義すべて入力済みで `done` になる

---

## 2. ターゲットセグメント

### 表示要件
- [ ] 既存セグメントをカード/テーブル形式で一覧表示する
- [ ] 各セグメントに「編集」「削除」ボタンを表示する
- [ ] 「セグメントを追加」ボタンを表示する

### 追加フォーム
| フィールド | 必須 |
|---|---|
| セグメント名 | ✓ |
| 規模 | |
| 特徴 | |

### インタラクション
- [ ] 追加: フォーム送信 → ID 生成 → 一覧に追加 → 自動保存
- [ ] 編集: インライン編集またはモーダル → 保存 → 自動保存
- [ ] 削除: 確認なしで即時削除・自動保存

## セクション完成度ロジック

```
status = 'empty'
  → 全サブセクションに入力なし

status = 'in_progress'
  → overview.definition が入力済み
    OR segments.length > 0
    OR consumerInsights.length > 0
    OR barriers.length > 0
    OR trends.length > 0

status = 'done'
  → overview の全フィールド入力済み
    AND segments.length >= 1
    AND (consumerInsights.length >= 1 OR barriers.length >= 1)
    AND trends.length >= 1
```

## エラーケース

| ケース | 期待動作 |
|---|---|

| localStorage への書き込み失敗 | エラーメッセージを通知 |
